# Street Music Voting Prototype

Прототип веб-приложения для уличного музыканта с двумя интерфейсами:
- публичная страница для зрителей (по QR),
- админка для музыканта.

## Стек

- Next.js (App Router)
- TypeScript
- In-memory store (без БД, для MVP)

## Быстрый старт

1. Установите Node.js 20+.
2. В корне проекта:
   - `npm install`
   - `npm run dev`
3. Откройте [http://localhost:3000](http://localhost:3000).

## Страницы

- `/` — публичный экран:
  - текущая песня (live),
  - ссылка на канал (mock),
  - текст о музыканте,
  - кнопки: голосование и донат.
- `/vote` — голосование:
  - поиск по песням,
  - лайки (можно выбрать несколько песен),
  - отправка выбора 1 раз за текущий раунд.
- `/admin` — интерфейс музыканта:
  - текущая песня,
  - выбор следующей вручную,
  - выбор случайной следующей,
  - завершение текущего раунда (переключение песни).

## Ограничение голосования с устройства (MVP)

Используется `localStorage`:
- ключ: `voted_round_<roundId>`;
- в рамках одного раунда устройство может отправить голосование один раз;
- после переключения раунда (`roundId` увеличивается) устройство снова может голосовать.

Важно: это базовая защита для прототипа, не антифрод.

## Логика раунда

- Текущая песня отображается на `/`.
- Зрители голосуют на `/vote`.
- В админке можно:
  - выбрать следующую вручную или случайно,
  - завершить текущую песню.
- При завершении:
  - текущая песня получает `playedCount +1`,
  - становится активной следующая,
  - счетчик голосов обнуляется для нового раунда,
  - `roundId` увеличивается.

## API (MVP)

- `GET /api/state` — текущее состояние.
- `POST /api/vote` — отправка голосов.
- `POST /api/admin/select-next` — выбрать следующую (`manual`/`random`).
- `POST /api/admin/advance-round` — завершить раунд и переключить текущую.

## Примечания

- Данные живут в памяти процесса (`lib/store.ts`) и сбрасываются после перезапуска сервера.
- Админка без авторизации (для прототипа).
